システム構成

Apache2 x Spring boot x MySQL によるポートフォリオ構成


概要

本ポートフォリオポータルは、Webサーバーとアプリケーションサーバーの責務を分離した構成で運用しています。
フロントエンドはApache2が担当し、バックエンド処理はSpring Bootアプリケーションが担当します。

全体構成

  • Webサーバー: Apache2
  • アプリケーションサーバー: Spring Boot (Java)
  • データベースサーバー: MySQL
  • 通信方式: HTTP（リバースプロキシ構成）
  • デプロイ形式: Spring Boot Jarをサーバー上で常駐実行

Apache2の役割

  • HTML / CSS / JavaScript などの静的コンテンツ配信
  • リクエストの受け口としての役割
  • 必要に応じて Spring Boot へのリクエスト転送

Webサーバーとしての役割を分離することで、アプリケーション側の負荷軽減と障害切り分けを容易にしています。

Spring Boot の役割

  • API エンドポイントの提供
  • 業務ロジックの実装
  • 今後の DB 連携・機能拡張の土台

フレームワークを利用することで、設定よりも実装に集中できる構成を意識しています。

この構成を選んだ理由

  • Webサーバーとアプリケーションサーバーの責務分離を理解するため
  • 実務で一般的な構成をポートフォリオに反映するため
  • 将来的なスケールや拡張を見据えた設計にするため

今後の拡張予定

  • API 機能の拡充
  • 認証・認可の実装
  • インフラ構成の自動化
  • AIとPostgreSQLを組み合わせた長期記憶参照テスト検証記録を追加
